Standlone Operation

Configuration

Overwhelmingly ZMAp is used in conjunctiion with Otterlace and Otterlace provides configuration to defined data sources etc and also some request data via the XRemote protocol. Data sources are specifed with a lengthy URL whcih include options for each source and some of these are common eg start and end coordinates, which are supplied by ZMap, and must be for fucntions such as 'Request from Mark'.

There are a few otehr arguments common to all scripts which relate to the sequence and for consistency, and to allow easy configuration for standalone operation it would be good to define these in another stanza:
sequence The name of the sequence eg 'chr6-18'
dataset eg 'human'
These could go into the [ZMap] stanza as currently do start and end coordinates, but perhaps a better solution would be ot create a stanza for each sequence of interest, with the name being the sequence name. These could go in a separate file in a similar manner to styles configuration.
