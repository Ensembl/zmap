Running ZMap on the Mac

From another PC

This is tricky as with the normal SSH route you have to set ip $DISPLAY to some magic value and the Mac refuses access to X.

On the ZMap Team's Mac

This is Gemma's PC (mac18480i) and if you log in you still have the same probelms with X.

Get Gemma to run ZMap and supply your own config: 

/nfs/users/nfs_m/mh17/zmap/ZMap/src/build/macosx/zmap --conf_dir=/nfs/users/nfs_m/mh17/.ZMap
You also have to give Gemma write permission on the directory and remove any existing zmap.log. Put the permissions back afterwards.

Using commands

Most unix commands appear to exist, but 'ls' does not work very well, you have to use 'echo *' instead.
